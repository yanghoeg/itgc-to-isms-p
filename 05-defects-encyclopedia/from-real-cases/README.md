# from-real-cases

작성자의 회계법인 IT 감사 실무에서 직접 본 결함 케이스. 모두 익명화된 형태로 기록한다.

## 익명화 규칙

| 항목 | 처리 |
|---|---|
| 회사명 | ABC회사 / DEF금융 / GHI제조 등 |
| 시스템명 | OOO ERP / XXX HR 등 |
| 인물명 | 모두 제거 (직책·역할만) |
| 날짜 | 연도까지만 또는 분기 단위 |
| 산업군 | 일반화 ("비금융 제조업" 등) |

## 작성 원칙

- 결함의 본질·발견 경로·근본 원인은 그대로 기록
- 빅4 메소돌로지의 구체적 절차는 기재 금지 (NDA 잠재 위반)
- 클라이언트 식별 가능 정보는 모두 제거
- 작성 후 commit 전 본인 검토 필수
