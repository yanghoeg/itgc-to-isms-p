# Program Development

ITGC 4번 영역 — 정보시스템 개발에 대한 통제. 신규 시스템 도입·개발 시 보안 요구사항 정의·검토·시험·이관 절차의 적정성 검증.

## 매핑되는 주요 ISMS-P 인증기준

- 2.8.1 보안 요구사항 정의
- 2.8.2 보안 요구사항 검토 및 시험
- 2.8.3 시험과 운영 환경 분리
- 2.8.4 시험 데이터 보안
- 2.8.5 소스 프로그램 관리
- 2.8.6 운영환경 이관

Change Management 영역과 인증기준이 상당 부분 겹친다. Development는 신규 도입·개발 시점, Change Management는 기존 시스템의 변경 시점으로 구분한다.
