# 05-defects-encyclopedia

결함 사례 백과. ISMS-P 인증기준별로 결함 패턴을 모은다.

## 두 가지 출처

- [`from-real-cases/`](./from-real-cases/) — 작성자의 회계법인 IT 감사 실무에서 직접 본 케이스. 회사명·시스템명·인물명·구체 날짜는 모두 익명화하여 결함의 본질만 남긴다.
- [`kisa-standard-cases/`](./kisa-standard-cases/) — KISA가 공개한 표준 결함사례. verbatim 인용 금지, 작성자의 말로 패러프레이징.

## 비교의 가치

같은 인증기준 아래 실무 사례와 KISA 표준을 같은 형식으로 정리하면, 결함 발견 패턴의 차이를 비교할 수 있다.

- 같은 패턴이면 → KISA 표준이 한국 IT 환경의 보편적 패턴을 잘 포착하고 있다는 증거
- 다른 패턴이면 → 실무 케이스가 KISA 표준을 보완하는 추가 자산
