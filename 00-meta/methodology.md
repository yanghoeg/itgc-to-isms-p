# 매핑 방법론

ITGC 감사 절차와 ISMS-P 인증기준을 어떤 원칙으로 매핑하는지 정리한다.

## 1. 매핑의 기본 단위

매핑의 최소 단위는 **ISMS-P 122개 인증기준 중 1개**다. ITGC 영역(Access / Change / Operations / Development) 단위로 묶지 않는다.

이유: ITGC 영역은 회계감사 관점의 분류이고, ISMS-P 인증기준은 정보보호 관점의 분류다. 1개 ITGC 영역이 여러 ISMS-P 인증기준에 흩어져 있고, 1개 ISMS-P 인증기준이 여러 ITGC 영역에 걸쳐 있다. 인증기준 단위로 매핑해야 양방향 추적이 가능하다.

## 2. 인증기준당 작성 항목

각 인증기준 페이지는 다음 6개 섹션으로 구성한다.

### 2.1. ISMS-P 인증기준 (요약)

KISA 안내서 본문을 직접 인용하지 않는다. 인증기준 번호·제목·핵심 통제 목표만 작성자의 말로 재서술한다.

```
번호: 2.5.1
제목: 사용자 계정 관리
핵심 통제 목표: (작성자가 재해석한 1~2 문장)
```

### 2.2. ITGC 매핑

이 인증기준이 ITGC 4대 영역 중 어디에 해당하는지 표기한다. 복수 영역에 걸칠 수 있다.

```
ITGC 영역: Access to Programs and Data
보조 영역: (해당 시) Computer Operations
```

### 2.3. 실무 사례 (익명화)

작성자가 회계법인 IT 감사 실무에서 직접 본 케이스를 익명화하여 기재한다. 회사명은 ABC회사 / DEF금융 / OOO ERP 등으로 일반화한다. 시스템 종류·산업군·결함의 본질만 남기고 식별 가능 정보는 모두 제거한다.

### 2.4. KISA 표준 결함사례 (요약)

KISA가 공개한 표준 결함사례를 작성자의 말로 패러프레이징한다. verbatim 인용 금지. 출처는 링크로 대체한다.

### 2.5. 시험 출제 패턴 (선택)

이 인증기준이 ISMS-P 인증심사원 시험에서 어떻게 출제되는지의 패턴을 정리한다. 작성자의 학습 노트 성격. 문제 풀이집은 만들지 않는다.

### 2.6. 관련 법령

이 인증기준이 어떤 법령 조항과 연결되는지 표기한다. 「개인정보 보호법」 제29조, 「정보통신망법」 제28조 등.

## 3. 매핑 우선순위

122개 인증기준을 한 번에 다 채우지 않는다. 우선순위는 다음과 같다.

1. **ITGC 실무 빈도가 높은 인증기준 우선** — 2.5 인증 및 권한 관리, 2.6 접근 통제, 2.8 정보시스템 도입 및 개발 보안, 2.9 시스템 및 서비스 운영
2. **결함 발견 빈도가 높은 인증기준 우선** — KISA 표준 결함사례 빈출 항목
3. **법령 직접 연결 인증기준 우선** — 「개인정보 보호법」 직접 조항 매핑
4. 나머지는 점진적으로 채운다.

## 4. 결함 사례 작성 규칙

`05-defects-encyclopedia/from-real-cases/` 에 추가하는 실무 사례는 다음 규칙을 따른다.

| 항목 | 규칙 |
|---|---|
| 회사명 | ABC회사 / DEF금융 / GHI제조 등으로 익명화 |
| 시스템명 | OOO ERP / XXX HR 등으로 익명화 |
| 인물명 | 모두 제거 (직책·역할만 표기) |
| 날짜 | 연도까지만 (예: "2024년") 또는 분기 단위 |
| 산업군 | 일반화 (예: "비금융 제조업", "금융 외 서비스업") |
| 결함의 본질 | 그대로 기록 — 이게 매핑의 핵심 자산 |

빅4 회계법인의 구체적 메소돌로지 절차는 NDA 잠재 위반 가능성으로 기재하지 않는다.

## 5. 1차 자료 처리 원칙

| 자료 | 인용 방식 |
|---|---|
| KISA ISMS-P 안내서 | 인증기준 번호·제목만. 본문 해석은 자체 작성. |
| 회계감사기준서 (315, 330, 402 등) | 기준서 번호·조항만. 본문 인용 금지. |
| 「개인정보 보호법」 등 법령 | 조문 번호 인용 가능. 조문 본문은 링크 또는 짧은 발췌 |
| 빅4 메소돌로지 | 인용 금지. NDA 잠재 위반 |
| 합격자 후기·블로그 | 2차 자료. 1차와 충돌 시 1차 우선 |

## 6. 매핑 검증

작성자가 매핑한 결과는 다음 방법으로 자가 검증한다.

1. **양방향 추적 가능성** — ITGC 영역 → 인증기준, 인증기준 → ITGC 영역 양방향으로 찾을 수 있는지
2. **KISA 표준 결함사례와의 정합성** — 실무 사례와 KISA 표준이 같은 인증기준 아래 같은 패턴으로 정렬되는지
3. **법령 조항과의 정합성** — 법령 직접 조항이 있는 인증기준은 그 조항이 명시되었는지

검증 실패 항목은 페이지 상단에 표기하고 차후 보완한다.
